# learn-csharp 约束速记

## 目标

- 这是一个 C# 学习课程项目，不是通用脚手架演示项目。
- 课程目标是让用户边读边做，正文优先承担主线教学和主线任务。
- 只有在正文之外仍然需要额外交付物时，才保留“实战”区。
- 默认用户画像是有 TypeScript / NestJS 背景的后端或全栈开发者。
- 讲解、命名对照、迁移路径、常见误区提示，都应优先利用 TS 用户熟悉的心智模型来帮助理解 C# / ASP.NET Core / EF Core / SignalR。

## 语言与文案

- 所有新增或修改文案使用简体中文。
- 文案优先直接、清楚、短句，不堆概念，不写空话。
- 不重复正文已经完整讲过并要求用户执行的步骤。
- “实战”不是装饰性标题，只在存在真实练习价值时使用。
- 在解释新概念时，优先说明它和 TypeScript / NestJS / Node.js 中对应物的相同点与关键差异。
- 对 TS 用户容易误判的点要主动提示，例如：`var` 不是 `any`、`record` 与 DTO 边界、`DbContext` 不是 Repository 同款替代、SignalR 与 Socket.IO 不兼容。

## 课程结构

- `LessonCheckpoint`：用于正文主线任务确认。
- `TeacherTask`：用于老师提示、阶段主线任务、边界提醒。
- `LessonStep`：只用于独立实战练习，不用于重复正文内容。
- `阶段验收问题`：用于检查理解，不替代实战。

## 实战保留规则

- 满足以下任一条件，才保留实战：
  - 需要组合多个知识点形成独立交付物。
  - 需要把正文知识迁移到新场景。
  - 需要完成复刻项目的阶段性任务。
- 出现以下情况，直接删除实战：
  - 只是把正文命令再做一遍。
  - 只是把正文代码再抄一遍。
  - 只是概念复述，没有额外交付物。
- 保留的实战应优先写成：任务目标、交付要求、验收点。
- 非必要不要保留大段重复代码示例。

## 当前课程层面的结论

- 环境准备与 C# 基础概念章节通常不需要额外实战区。
- Setup、类型系统、LINQ、async、Controller CRUD、OpenAPI 基础配置这类内容，如果正文已经完成主线任务，则不再额外保留重复练习。
- EF 关系建模、事务、SignalR、测试、可观测性、部署这类跨知识点阶段任务，可以保留实战，但必须是独立交付物导向。

## 代码修改原则

- 优先复用现有组件、工具函数和课程结构。
- 不凭空发明 API、类型、目录约定。
- 不手写或生成无依据的 `d.ts`。
- 修改范围尽量局部，避免无关重构。
- 命名遵循现有项目风格。

## 前端约束

- 继续使用现有 React + TypeScript + Ant Design + Tailwind 组合。
- 新组件优先放在现有 `src/components` 体系内。
- 只有在现有组件无法表达时才新增结构。
- 不为“看起来完整”而添加无必要 UI。

## 验证规则

- 改动代码后，结束前至少执行：
  - `yarn lint`
  - `yarn build`
- 只有验证通过后，才能声称改动完成。

## 禁止项

- 不新增无意义实战。
- 不保留与正文重复的练习。
- 不因为内容多就默认用户需要更多任务。
