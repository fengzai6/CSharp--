# 04 - 学生体验优化

## 目标

从学生学习的角度优化网站交互，让学习路径更清晰、进度更直观。

## 必读文档

- [00-overview.md](00-overview.md) — 总览与共识
- [02-restore-content.md](02-restore-content.md) — 内容补全结果

## 任务详情

### 候选优化项（实施前与用户确认范围）

1. **学习路径总览页/入口**
   - docs/README.md 有完整学习路径和"5 个思维差异"，但网页缺失
   - 考虑新增首章总览或在侧边栏顶部呈现

2. **进度统计可视化**
   - 当前只有 checkbox 打卡，缺乏整体进度感知
   - 考虑在侧边栏或助手区显示"已完成 X/19 节"

3. **章节内快速跳转**
   - 长内容需要章节内锚点导航（当前 `course-assistant` 已有"本章目录"，确认是否够用）

4. **代码块体验**
   - 确认 `CodeCopy` 复制功能正常
   - 长代码块考虑是否需要折叠

### 约束

- 不新增依赖
- 复用现有组件和 Zustand store
- 进度相关数据通过现有 store 派生，不重复存储

### 验证标准

- [ ] 优化项与用户确认后实施
- [ ] 进度统计准确反映实际完成情况
- [ ] `yarn lint` 通过
- [ ] `yarn build` 通过
