# 04 - 学生体验优化

## 目标

从学生学习的角度优化网站交互，让学习路径更清晰、进度更直观，流程更自然好用。

## 必读文档

- [00-overview.md](00-overview.md) — 总览与共识
- [02-restore-content.md](02-restore-content.md) — 内容补全结果

## 任务详情

### 实施范围

1. **学习驾驶舱**
   - 将侧边栏顶部改为学习驾驶舱，展示当前学习位置、总完成度、继续学习入口
   - 模块列表展示学习路径顺序、章节目标、完成数、当前/已完成/未开始状态

2. **正文学习状态条**
   - 在课程正文顶部展示当前模块序号、当前小节序号、总进度、下一步方向
   - 优化"标记完成"相关文案，让它更贴近学习动作

3. **当前章学习助手**
   - 将右侧助手改为当前章学习助手，优先展示本章目标、本章进度、本章目录
   - 保留老师提示、常见误区、阶段验收，但按学生学习顺序重新组织

4. **完成后的下一步引导**
   - 学完一节后优先引导"标记完成并继续"
   - 最后一节提供复习入口，并保持学习工具风格

5. **色系调整**
   - 从偏亮的蓝绿色大面积背景调整为更沉稳的蓝灰/浅 slate 背景
   - teal 只用于主按钮、进度条、当前状态和少量强调
   - 保持可读性和学习工具的克制感，不使用装饰性图片或新增依赖

6. **代码块体验**
   - 保持现有 `CodeCopy` 复制和高亮能力
   - 本任务不新增长代码折叠，避免打断连续阅读

### 约束

- 不新增依赖
- 复用现有组件和 Zustand store
- 进度相关数据通过现有 store 派生，不重复存储
- 不新增测试文件，使用 `yarn lint` 和 `yarn build` 做基础验证

### 验证标准

- [x] 学习路径、当前进度、下一步动作在首屏区域更清晰
- [x] 进度统计准确反映实际完成情况
- [x] 右侧助手能体现当前章目标、目录、误区和验收问题
- [x] 色系更有活力且正文可读性不下降
- [x] `yarn lint` 通过
- [x] `yarn build` 通过

## 结果

已完成学生体验优化，`yarn lint` 和 `yarn build` 通过。

- 侧边栏改为学习驾驶舱：展示当前学习位置、总完成度、继续学习入口，并在模块列表中展示目标、完成数、当前/已完成/未开始状态。
- 正文顶部新增学习状态条：展示当前位置、下一步方向和总进度；本节完成按钮文案改为更贴近学生学习动作。
- 右侧助手改为当前章学习助手：展示本章目标、本章进度、本章目录，并保留学习提示、容易踩坑、学完自查和复习策略。
- 新增 `getCourseProgress`、`getModuleProgress`、`getSectionPosition` 工具函数，所有进度均从现有课程数据和 `completedSectionIds` 派生。
- 色系从高亮浅青收敛为蓝灰/浅 slate 背景 + teal 强调，同时更新 Ant Design 主题 token 和课程正文局部样式。
- 保持 `CodeCopy` 复制与高亮能力，不新增长代码折叠。

## 决策与备注

- **2026-06-05 学习体验方向**：将网站从“章节列表 + 正文阅读”调整为“学习驾驶舱 + 当前章助手 + 下一步引导”，让学生始终知道现在在哪、学完了多少、下一步做什么。
- **2026-06-08 色系方向**：采用更沉稳的蓝灰/浅 slate 作为大面积背景，teal 用于主按钮、进度条、当前状态和少量强调；不新增装饰图片或依赖。
