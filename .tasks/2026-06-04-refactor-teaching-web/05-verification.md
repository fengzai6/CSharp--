## 目标

全流程测试学习网站，确保从老师和学生的立场都达到可用标准。

## 任务详情

### 验证维度

#### 1. 老师视角

- [ ] 每章有清晰的学习目标（对应 docs 中的"本阶段只解决 X 个问题"）
- [ ] 老师提示、常见误区、验收问题在助手区正确显示
- [ ] 内容与 docs 原文一致，没有过度摘要或信息丢失
- [ ] 代码示例有注释，说明关键概念
- [ ] 每节课结尾有明确的下一步引导

#### 2. 学生视角

- [ ] 内容阅读流畅，不僵硬
- [ ] 练习打卡能正常工作，进度本地保存
- [ ] 代码复制功能正常
- [ ] 章节导航清晰，能快速跳转
- [ ] "继续学习"能正确恢复到上次位置

#### 3. 技术质量

- [ ] `yarn lint` 通过
- [ ] `yarn build` 通过
- [ ] 开发服务器启动正常
- [ ] 无 console 报错
- [ ] 所有 19 节课能正常显示

### 全流程测试场景

1. **新学生首次访问**
   - 打开网站 → 查看首章"环境准备与项目骨架" → 完成第一个练习打卡 → 刷新页面，验证进度保存

2. **老学生继续学习**
   - 已完成 3 章 → 点击"继续学习" → 正确跳转到第 4 章第 1 节

3. **跨章节跳跃**
   - 从第 1 章跳到第 5 章 → 侧边栏正确展开 → 助手区显示第 5 章的老师提示

4. **代码复制**
   - 随机选 5 个代码块 → 点击复制 → 粘贴验证

5. **进度重置**
   - 完成 10+ 个打卡 → 点击"重置进度" → 确认清空 → 验证所有打卡重置
