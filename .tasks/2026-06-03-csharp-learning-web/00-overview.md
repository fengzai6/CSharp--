# C# 学习网页

## 目标

把 `docs` 中的 C#/.NET 学习文档做成可逐步学习的本地网页，支持章节导航、代码复制、练习打卡和本地进度保存。

## 背景与约束

- 项目目录：`learn-csharp`
- 技术栈：React 19、Vite、TypeScript、Tailwind CSS、Ant Design、Zustand
- 文档来源：`docs/*.md`
- 当前要求：
  - 使用简体中文界面和说明。
  - 优先复用现有依赖，不新增依赖，除非计划确认后确有必要。
  - Zustand 本地持久化必须使用 `persist` 中间件，不手写 localStorage 持久化。
  - 课程内容应以 `docs` 文档为主，不做过度摘要或脱离原文的硬编码课程。
  - 网页应补充学习目标、老师提示、验收问题等教学辅助内容。
  - 可以拆分组件，避免单个文件过大。
- 测试约束：
  - 新增测试文件或扩大测试范围前需要用户确认。
  - 本阶段默认使用 `yarn lint` 和 `yarn build` 做基础验证。

## 执行顺序

1. 先清理当前临时实现，避免在错误结构上继续扩展。
2. 再优化 `docs`，因为网页正文应来自文档原文。
3. 然后建立 Markdown 内容模型，把原文、代码块、checklist 和教学增强映射到网页。
4. 再实现 Zustand `persist` 进度存储。
5. 最后实现拆分后的学习 UI，并做验证审查。

## 共识与决策

- **2026-06-03 持久化方案**：使用 Zustand `persist` 中间件保存学习进度，避免手写 localStorage 同步逻辑。
- **2026-06-04 内容方案**：课程正文不使用 block 数据驱动渲染。每节课应是独立 React 组件，直接抄写/扩展原文内容，并在 JSX 中自由插入老师任务、检查点和下一步交互。
- **2026-06-03 教学方案**：在原文基础上补充老师提示、学习目标、验收问题，帮助按阶段学习和自测。
- **2026-06-03 组件拆分方案**：允许按布局、Markdown 渲染、代码复制、进度面板、教学辅助面板拆分组件，避免单个文件过大。
- **2026-06-03 执行限制**：当前只完善任务计划，不执行实现。
- **2026-06-03 下一步交互**：每个阶段结尾必须提供下一步入口，最后一章提供回到总览或复习入口。
- **2026-06-04 组件方案**：只保留导航、进度、布局等必要元信息；课程正文由 `lessons/<module>/<lesson>.tsx` 这类组件承载。
