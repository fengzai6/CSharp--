# 02 - 优化 docs 教学内容

## 目标

在不偏离原学习路线的前提下，优化 `docs` 文档，使其更适合“老师式”逐步学习。

## 必读文档

- [00-overview.md](00-overview.md) — 总览与共识
- [docs/README.md](../../docs/README.md) — 学习路径总览
- [docs/00-环境准备与项目骨架.md](../../docs/00-环境准备与项目骨架.md) — 阶段 00
- [docs/01-CSharp语言核心.md](../../docs/01-CSharp语言核心.md) — 阶段 01
- [docs/02-ASPNET-Core框架.md](../../docs/02-ASPNET-Core框架.md) — 阶段 02

- [docs/06-工程化与进阶.md](../../docs/06-工程化与进阶.md) — 阶段 06

## 任务详情

优化方向：

1. 技术准确性
   - 修正自相矛盾或过时内容。
   - 标注版本边界，例如 .NET 8、.NET 9、.NET 10、EF Core 10。
   - 不凭空生成 API；不确定的 API 需查官方文档或保守表述。
2. 老师式学习引导
   - 每个阶段补充“本章你要掌握什么”。
   - 每个阶段补充“学习顺序建议”。
   - 每个阶段补充“常见误区”。
   - 每个阶段补充“阶段验收问题”。
3. 实战连贯性
   - 每章练习尽量落到同一个复刻项目中。
   - 练习要能从简单到复杂递进。
   - 明确哪些练习是必须完成，哪些是扩展。
4. 网页适配
   - 保留 fenced code block，便于网页识别并提供复制。
   - checklist 使用标准 `- [ ]` 写法，便于网页持久化。
   - 表格不要过宽；必要时拆分或简化列。

## 不做事项

- 不改变学习路线的大方向。
- 不把文档改成营销文案。
- 不删除关键概念，只优化表达、顺序和验收。

## 结果

已为 7 份 docs 文档补充老师式学习引导：

- 本章你要掌握什么
- 老师提示
- 学习顺序建议
- 常见误区
- 阶段验收问题

同时保留原学习路线、代码块和 checklist 格式，便于后续网页渲染与进度记录。

## 决策与备注

不改变学习路线，只补充教学辅助内容和阶段验收。
